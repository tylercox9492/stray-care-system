# Project Structure

This project is organized into three main components: backend, database, and frontend.

## Backend (`/backend`)

The backend is a Spring Boot application that follows a layered architecture:

- `controller/` – REST API endpoints (e.g., authentication, dogs, sightings, admin)
- `service/` – Business logic (e.g., Cloudinary uploads, QR code generation)
- `repository/` – JPA repositories for database access
- `model/` – Entity classes representing database tables
- `config/` – Security and configuration (JWT authentication, Spring Security)

**Additional backend components:**

- `application.properties` – Configuration for database, Cloudinary, etc.
- `pom.xml` – Maven dependencies and build configuration
- `target/` – Compiled output (auto-generated by Maven)

## Database (`/database`)

- `schema.sql` – SQL script used to initialize the MySQL database schema

## Frontend (`/frontend`)

The frontend consists of static HTML, CSS, and JavaScript files:

### HTML Pages

- `index.html` – Landing page
- `dog_profile.html` – Dog details and map view
- `sighting_form.html` – Submit sightings with GPS and photo upload
- `volunteer_dashboard.html` – Heatmap and filtering tools
- `login.html` – Authentication interface

### JavaScript

- `maps.js` – Google Maps integration (markers, heatmap)
- `config.js` – Configuration (e.g., API keys)

### Styling

- `styles.css` – Shared styles

### Assets

- `images/` – Static images and UI assets

## How to Run the Project

### 1. Start the Database and Backend with Docker Compose

This repository includes a `docker-compose.yml` that starts:

- MySQL 8.4
- the Spring Boot backend
- the static frontend through Nginx

The MySQL container automatically runs `database/schema.sql` the first time the database volume is created, so you do not need to create the schema manually.

Setup:

1. Copy `.env.docker.example` to `.env.docker`
2. Fill in the placeholder values in `.env.docker`
3. Start the stack:

```bash
docker compose --env-file .env.docker up --build
```

Services:

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8080`
- MySQL: `localhost:3306`

Notes:

- If you change `database/schema.sql` and want MySQL to re-run it from scratch, remove the `mysql_data` volume before starting again
- The backend now reads database credentials from environment variables first, with local defaults as a fallback

### 2. Configure the Frontend for Manual Local Runs

Create a local `frontend/config.js` file containing a Google Maps API key, defined as a global configuration object accessible by the frontend scripts. This file is ignored by Git and should stay local:
```javascript
window.APP_CONFIG = {
    GOOGLE_MAPS_API_KEY: "your_api_key_here",
    API_BASE_URL: "http://localhost:8080/api"
};
```

If you are using Docker Compose for the frontend too, you do not need this file inside the container. The container generates its own `config.js` from `.env.docker`.

### 3. Run the Frontend Manually

From the `frontend/` directory, start a simple local server:
```bash
python -m http.server <port>
```

Then open in your browser:
```
http://localhost:<port>
```

### 4. Manual Backend Setup Without Docker

If you prefer to run MySQL and Spring Boot directly on your machine:

1. Create a MySQL database such as `stray_care_system`
2. Run the schema script manually:

```sql
SOURCE database/schema.sql;
```

3. Copy `backend/.env.example` to `backend/.env`
4. Export those values into your shell or IDE run configuration before starting the backend
5. From `backend/`, run:

```bash
mvn spring-boot:run
```

Example PowerShell session:

```powershell
$env:SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/stray_care_system"
$env:SPRING_DATASOURCE_USERNAME="root"
$env:SPRING_DATASOURCE_PASSWORD="your_database_password_here"
$env:CLOUDINARY_CLOUD_NAME="your_cloud_name_here"
$env:CLOUDINARY_API_KEY="your_cloudinary_api_key_here"
$env:CLOUDINARY_API_SECRET="your_cloudinary_api_secret_here"
```
